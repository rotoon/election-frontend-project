# Election Frontend

ระบบ Frontend สำหรับการเลือกตั้งและรายงานผลแบบภาษาไทย พัฒนาด้วย Next.js โดยรวมทั้งหน้าสาธารณะสำหรับดูผลเลือกตั้ง, ฝั่งผู้มีสิทธิ์เลือกตั้ง, ฝั่งผู้ดูแลระบบ และฝั่งเจ้าหน้าที่ กกต. ไว้ในโปรเจกต์เดียว

---

## ภาพรวมโปรเจกต์

โปรเจกต์นี้คือเว็บแอปสำหรับจัดการและแสดงข้อมูลการเลือกตั้งแบบครบวงจร โดยแบ่งการใช้งานออกเป็นหลายบทบาทและหลายมุมมอง ได้แก่

- **หน้าสาธารณะ** สำหรับดูผลเลือกตั้ง ภาพรวม รายเขต พรรค และผู้สมัคร
- **ผู้มีสิทธิ์เลือกตั้ง (Voter)** สำหรับเข้าสู่ระบบ ดูข้อมูลส่วนตัว และลงคะแนน
- **ผู้ดูแลระบบ (Admin)** สำหรับจัดการผู้ใช้งานและเขตเลือกตั้ง
- **เจ้าหน้าที่ กกต. (EC)** สำหรับจัดการพรรค ผู้สมัคร และควบคุมสถานะการเลือกตั้ง

---

## โปรเจกต์นี้มีอะไรบ้าง

### 1) หน้าสาธารณะ (Public Pages)

ใช้สำหรับให้บุคคลทั่วไปเข้ามาดูข้อมูลการเลือกตั้งได้โดยไม่ต้องล็อกอิน

- **หน้าแรก `/`**
  - แสดงภาพรวมผลเลือกตั้ง
  - แสดงพรรคอันดับต้น
  - แสดงความคืบหน้าการนับคะแนน
  - มี sidebar สำหรับนำทางไปยังหน้าสำคัญ

- **หน้าผลรายเขต `/district`**
  - เลือกภูมิภาค / จังหวัด / เขตเลือกตั้ง
  - ดูผลการเลือกตั้งรายเขต
  - รองรับการใช้งานทั้งเดสก์ท็อปและมือถือ

- **หน้าพรรคการเมืองแบบสาธารณะ `/parties-list`**
  - แสดงรายชื่อพรรคทั้งหมด
  - มีโลโก้ ชื่อพรรค และนโยบายแบบย่อ
  - เปิดดูรูปหรือรายละเอียดเพิ่มเติมได้

- **หน้าผู้สมัครแบบสาธารณะ `/candidates-list`**
  - แสดงรายชื่อผู้สมัครทั้งหมด
  - ค้นหาและกรองตามพรรค จังหวัด และเขตได้
  - เปิดดูรายละเอียดผู้สมัครใน dialog ได้

---

### 2) ฝั่งผู้มีสิทธิ์เลือกตั้ง (Voter)

เป็นส่วนสำหรับผู้ใช้งานที่ล็อกอินเข้ามาเพื่อใช้สิทธิ์และดูข้อมูลของตัวเอง

- **หน้าลงคะแนน `/vote`**
  - แสดงผู้สมัครในเขตของผู้ใช้
  - ตรวจสอบสถานะเปิด/ปิดหีบ
  - เลือกผู้สมัครและยืนยันการลงคะแนน
  - รองรับการแก้ไขคะแนนที่เคยเลือกไว้

- **หน้าโปรไฟล์ `/profile`**
  - ดูข้อมูลส่วนตัว
  - ดูจังหวัด อำเภอ เขตเลือกตั้ง
  - ดูสถานะการเลือกตั้งในเขตของตัวเอง

---

### 3) ฝั่งผู้ดูแลระบบ (Admin)

ใช้สำหรับงานบริหารข้อมูลหลักของระบบ

- **จัดการเขตเลือกตั้ง `/admin/constituencies`**
  - ดูรายการเขตเลือกตั้ง
  - กรองข้อมูลตามจังหวัด
  - เพิ่ม แก้ไข และลบเขตเลือกตั้ง
  - ดูรายชื่อผู้สมัครในแต่ละเขต

- **จัดการผู้ใช้งาน `/admin/users`**
  - ค้นหาและกรองผู้ใช้
  - ดูรายการผู้ใช้ทั้งแบบ desktop และ mobile
  - เปลี่ยนบทบาทของผู้ใช้ในระบบ

---

### 4) ฝั่งเจ้าหน้าที่ กกต. (EC)

ใช้สำหรับงานปฏิบัติการที่เกี่ยวข้องกับการเลือกตั้งโดยตรง

- **จัดการพรรคการเมือง `/ec/parties`**
  - เพิ่ม แก้ไข และลบพรรค
  - อัปโหลด/แสดงโลโก้พรรค
  - ดูรายการพรรคทั้งหมด

- **จัดการผู้สมัคร `/ec/candidates`**
  - เพิ่ม แก้ไข และลบผู้สมัคร
  - ค้นหา กรอง และเรียงลำดับข้อมูล
  - ดูรายละเอียดและรูปภาพของผู้สมัคร

- **ควบคุมการเลือกตั้ง `/ec/control`**
  - เปิด/ปิดหีบรายเขต
  - เปิด/ปิดหีบหลายเขตหรือทุกเขต
  - ดูรายชื่อผู้สมัครในแต่ละเขต
  - ใช้สำหรับควบคุมสถานะการเลือกตั้งในระบบ

---

## โครงสร้างโปรเจกต์

```text
app/           หน้าต่าง ๆ ของระบบด้วย Next.js App Router
components/    UI components และ feature components
hooks/         data fetching, query, mutation, custom hooks
lib/           utility, API client, error handling, transform functions
store/         Zustand store สำหรับ auth state
types/         type definitions ของแต่ละ domain
docs/          เอกสาร API และข้อมูลอ้างอิง
public/        static assets
```

---

## หมวดของโค้ดหลัก

### Routing และ Layout
- ใช้ **Next.js App Router**
- มี layout แยกตามกลุ่มผู้ใช้งาน
- มี error boundary และ loading state แยกในหลายส่วนของระบบ

### Components
- มีทั้ง **UI primitives** และ **feature-specific components**
- ใช้แนวทาง reusable components
- มี shared components สำหรับ dialog, pagination, sidebar layout และ loading

### Data Layer
- ใช้ **TanStack React Query** สำหรับ server state
- ใช้ **Axios** เป็น API client กลาง
- ใช้ **hooks/** เป็นจุดรวมของ query และ mutation
- หลีกเลี่ยงการเรียก API ตรงจาก component

### State Management
- ใช้ **Zustand** สำหรับ auth/session state
- ใช้ local state สำหรับ UI state ทั่วไป
- มีการ persist auth state บางส่วนไว้ฝั่ง client

### Type System
- ใช้ **TypeScript strict mode**
- แยก type ตาม domain เช่น auth, user, party, candidate, constituency, vote, dashboard
- มีชั้น transform สำหรับแปลงข้อมูลจาก backend ให้เหมาะกับ UI

## การยืนยันตัวตนและสิทธิ์การเข้าถึง

ระบบรองรับหลายบทบาทของผู้ใช้

- `ROLE_VOTER`
- `ROLE_EC`
- `ROLE_ADMIN`

แนวทางทำงานโดยรวม:
- เข้าสู่ระบบผ่าน `/auth`
- เก็บ token สำหรับเรียก API
- ใช้ middleware ป้องกัน route ตาม role
- ถ้ามีหลายบทบาท สามารถเลือกใช้งานผ่าน `/portal`

## จุดเด่นของโปรเจกต์

- รองรับหลายบทบาทในระบบเดียว
- มีทั้งหน้า public dashboard และหน้าจัดการหลังบ้าน
- แยกโครงสร้างไฟล์ชัดเจน อ่านและขยายต่อได้ง่าย
- ใช้ modern frontend stack
- รองรับ responsive UI
- มี dialog, loading state, toast และ feedback ครบ
- เตรียม test infrastructure ไว้แล้ว

## เหมาะกับใคร

โปรเจกต์นี้เหมาะสำหรับ:

- นักพัฒนาที่ต้องการระบบตัวอย่างแบบ **multi-role dashboard**
- ทีมที่กำลังทำระบบเลือกตั้ง ระบบลงคะแนน หรือระบบ workflow ภาครัฐ
- ผู้ที่ต้องการตัวอย่างโครงสร้าง **Next.js + React Query + Zustand + TypeScript** ที่แยก concern ค่อนข้างชัด

---

## สรุป

โปรเจกต์นี้คือ **Election Frontend** ที่รวมทั้ง

- ระบบดูผลเลือกตั้งสำหรับสาธารณะ
- ระบบใช้งานของผู้มีสิทธิ์เลือกตั้ง
- ระบบบริหารของแอดมิน
- ระบบปฏิบัติการของเจ้าหน้าที่ กกต.

ไว้ในโค้ดเบสเดียว โดยออกแบบให้มีโครงสร้างชัดเจน แยกหน้าที่ของแต่ละส่วนดี และพร้อมต่อยอดได้ทั้งเชิงฟีเจอร์และเชิงสถาปัตยกรรม
